## Quick summary

- Mandatory base environment is `production`; `preview` and `development` are explicit project-level opt-in environments.
- A project has multiple services and multiple environments.
- Services inherit environment scope from their parent project.
- Additional custom environment variants are supported and configurable.
- Multi-node production baseline targets Docker Swarm manager/worker orchestration with Traefik-first ingress.
- Platform target aligns with modern self-hosted PaaS expectations (Coolify/Dokploy-class capabilities).

## Why this exists

- Keep one clear baseline for product and architecture decisions.
- Prevent contradictions across feature docs.
- Make migrations and implementation validation easier.
